# Инструкция по учёту доходов ООО «Орбо» в Контур.Эльбе

УСН 6% (доходы), кассовый метод. Документ описывает источники наших доходов, какие документы по ним формируются автоматически, и как разметить поступления эквайринга в Эльбе вручную.

---

## 1. Что считается нашим доходом

| Тип дохода | Источник | Размер | Документ-основание |
|---|---|---|---|
| **Сервисный сбор** | Платежи участников событий (через эквайринг T-Bank) | 10% — для физлиц-организаторов, 5% — для юрлиц/ИП | АУ-N — акт об оказании услуг (на «Розничные покупатели») |
| **Агентское вознаграждение** | Удерживается из платежей участников при работе с организаторами-юрлицами/ИП | 5% от тела билета | АВ-N — УПД на каждого организатора-юрлицо ежемесячно |
| **Лицензионные платежи (тарифы)** | Оплата платных тарифов Орбо организациями | По прайсу тарифа (1 500 ₽, 5 000 ₽ и т.п.) | АЛ-N — акт передачи неисключительных прав (один на каждый платёж) |
| **Ускоренная проверка реквизитов** | Разовый платёж за быстрое подтверждение реквизитов организатора | 200 ₽ | АЛ-N — отдельный акт по аналогии с тарифом |

### Что НЕ считается нашим доходом
- **Тело билета** — это деньги принципала-организатора. Мы получаем их на расчётный счёт в составе зачислений эквайринга, но удерживаем для последующего вывода организатору.
- **Возвраты тела билета участникам** — не наш расход (это не наши деньги). Если организатору пришлось вернуть, мы лишь технически делаем перевод обратно.
- **Выплаты (выводы) принципалам** — расход с расчётного счёта, но не наш расход для УСН (мы не имеем права уменьшать доход на эту сумму).

## 2. Где смотреть наши доходы

В админке: **Суперадмин → Бухгалтерские документы** (`/superadmin/accounting`).

Сверху страницы — блок **«Книга доходов ОРБО (УСН 6%)»**.

### Что в нём есть

1. **Фильтр по периоду** — выбираете «с» и «по» (можно произвольный диапазон).
2. **Чекбокс «Показать тестовые»** — по умолчанию выключен. Тестовые транзакции, помеченные как такие, не входят в книгу доходов.
3. **Сводка** — итог за период, разбитый по 5 категориям (сервисный сбор, агентская комиссия, тарифы, возвраты сборов, возвраты комиссии).
4. **Таблица «По дням»** — построчная разбивка для каждого дня периода.
5. **Раздел «Детализация»** (раскрывается по клику) — каждая отдельная транзакция за период с возможностью пометить её тестовой.

> **Пометка транзакции как тестовой:** в строке детализации справа есть кнопка-колбочка 🧪. Клик по ней помечает транзакцию `is_test=true` (попадает в подсветку оранжевым) и она сразу исчезает из сводки, если чекбокс «Показать тестовые» выключен. Повторный клик — снимает метку. Подходит для редких случаев тестов терминала. Кнопка работает только для транзакций с привязанной платёжной сессией; для лицензионных платежей-инвойсов без сессии разметка делается разработчиком на запрос.

### Кнопки выгрузки

| Кнопка | Файл | Когда использовать |
|---|---|---|
| **Книга доходов** | `orbo-income-ledger_*.xlsx` | Главный отчёт для бухгалтера. 3 листа: «Сводка по дням», «Детализация», «О документе». |
| **Реестр транзакций** | `orbo-transactions_*.xlsx` | Полный список всех финансовых операций (в том числе и не наш доход) — для камеральной налоговой проверки. |
| **Реестр выводов** | `orbo-withdrawals_*.xlsx` | Заявки на вывод средств организаторам с реквизитами банка и номерами актов. |
| **Реестр возвратов** | `orbo-refunds_*.xlsx` | Возвраты тела билета участникам и реверсы агентских комиссий. |

## 3. Как разметить поступления эквайринга в Эльбе

Эквайринг (T-Bank или CloudPayments — каждый платит на свой расчётный счёт) **ежедневно утром** зачисляет на счёт сумму вчерашних оплат **за вычетом комиссии шлюза**. Эта сумма — смешанная: в ней и наш доход (сервисный сбор + агентская комиссия + лицензионные платежи), и средства принципалов (тело билетов, идущие на выводы).

> **Важно:** все платежи в Орбо проходят через эквайринг как обычные оплаты картой — и от участников событий, и от организаторов за тариф/проверку реквизитов. Отдельных банковских переводов от организаций-юрлиц по тарифам нет.
>
> В период миграции платформа использует **два эквайринга**: T-Bank и CloudPayments. Каждый из них зачисляет на **свой** расчётный счёт независимо от другого. Постепенно мы переключим всех на CloudPayments (комиссия ниже), а T-Bank оставим как резерв; до этого размечать в Эльбе нужно оба зачисления.

В Эльбе при ручном вводе на каждое поступление есть поле **«Сумма дохода УСН»**. Туда нужно вписывать **только нашу часть**.

### Пошаговый алгоритм для каждого зачисления эквайринга

1. **Узнайте дату операционного дня** — это вчерашний день, за который пришло зачисление сегодня утром, и **через какой шлюз** оно пришло (T-Bank или CloudPayments — определяется по банку-отправителю в выписке).
2. **Откройте «Книгу доходов»** в админке Орбо за этот день (фильтр «с=…» и «по=…» — одна и та же дата).
3. В таблице «По дням» посмотрите столбец **«через T-Bank»** или **«через CloudPayments»** — это сразу даёт нашу часть, пришедшую через **именно этот шлюз** за этот день. Это и есть число для поля «Сумма дохода УСН» на этом банковском поступлении.
4. **Остальную часть** зачисления (то есть «зачисление от эквайринга» − «наш доход через этот шлюз») в Эльбе помечайте как «не доход» либо в комментарии указывайте «средства принципалов, агентская схема».
5. Если в один день были зачисления от обоих шлюзов — повторите для каждого. Они НЕ суммируются: у Эльбы это два отдельных банковских поступления.

### Сверка
- **Итог в Книге доходов за день для конкретного шлюза** (столбец «через T-Bank» / «через CloudPayments») = (сервисный сбор + агентская комиссия + лицензионные платежи), пришедшие через этот шлюз.
- **Сумма в зачислении эквайринга от этого шлюза** = (тело билетов всех платежей вчера через этот шлюз) + (наша часть через этот шлюз) − (комиссия шлюза, удержанная при зачислении).
- Если расхождение — открыть «Реестр транзакций» в админке (там тоже есть столбец «Шлюз»), отфильтровать по дате и шлюзу, сверить отдельные платежи.

### Где увидеть разрез по шлюзам
- В сводке (вверху страницы) — карточки «через T-Bank: …», «через CloudPayments: …».
- В таблице «По дням» — отдельные синие столбцы для каждого шлюза.
- В выгрузке **Книга доходов (XLSX)** — на листе «Сводка по дням» эти же столбцы; на листе «Детализация» добавлен столбец «Шлюз» в каждой строке.
- В выгрузке **Реестр транзакций (XLSX)** — столбец «Шлюз».

### Особый случай: возвраты участникам
Если вчера T-Bank сделал возврат участнику (refund по карте), то в зачислении эквайринга соответствующая сумма будет отрицательной (или вообще зачисление меньше). В Книге доходов в этот день в категории «Возвраты сборов» / «Возвраты комиссии» **могут стоять отрицательные суммы**, которые уменьшают наш доход. По умолчанию мы возвращаем только тело билета, наш сервисный сбор остаётся (см. договор-оферта, п. о возвратах). Если в исключительных случаях возвращался и сервисный сбор — он отразится в этой категории со знаком минус.

## 4. Лицензионные платежи и проверка реквизитов

Лицензионные платежи (тарифы и ускоренная проверка реквизитов) сейчас принимаются **через тот же эквайринг T-Bank, что и платежи участников за билеты** — оплата картой, как от физлица, даже если договор с организацией-юрлицом. Отдельных банковских переводов на расчётный счёт по этим платежам нет.

- **Тариф Pro / Club** (1 500 ₽, 5 000 ₽ и т.п.) — оплата подписки.
- **Ускоренная проверка реквизитов** (200 ₽) — разовый платёж при заключении договора.

После того, как платёж подтверждён через эквайринг (или суперадмин фиксирует его вручную):
- Создаётся `org_invoices` запись со статусом `paid`
- Автоматически генерируется акт **АЛ-N** (передача неисключительных прав / разовая лицензия)
- Акт автоматически отправляется в Контур.Эльбу
- Сумма платежа попадает в Книгу доходов за день в категории «Лицензионные платежи» — **100% наш доход**, разметка отдельной строкой не нужна, она уже учтена в дневной сумме раздела 3.

## 5. Документы, отправляемые в Эльбу автоматически

| Документ | Когда формируется | Куда отправляется |
|---|---|---|
| **АЛ-N** (Акт лицензии) | На каждую оплату тарифа или проверки реквизитов, в момент подтверждения оплаты | Эльба, контрагент = плательщик |
| **АВ-N** (УПД на агентское вознаграждение) | Раз в месяц для каждого организатора-юрлица с ненулевой комиссией | Эльба, контрагент = организатор-юрлицо |
| **АУ-N** (Акт услуг розничным покупателям) | Раз в месяц/период по сервисным сборам, генерируется суперадмином из админки | Эльба, контрагент = «Розничные покупатели» |
| **ОА-N** (Отчёт агента) | Раз в месяц для каждого организатора | Не отправляется в Эльбу — отдаётся принципалу как отчёт по ст. 1008 ГК РФ |

## 6. КУДиР (книга учёта доходов и расходов)

Эльба формирует КУДиР автоматически на основе зачислений с проставленной «суммой дохода УСН». Если все зачисления размечены корректно по этой инструкции, КУДиР будет верной.

Для камеральной налоговой проверки достаточно:
- КУДиР из Эльбы
- «Книга доходов» (XLSX) из нашей админки за тот же период — как первичная расшифровка
- «Реестр транзакций» (XLSX) — детализация если налоговая попросит

## 7. Что делать с тестовыми транзакциями

Тестовые транзакции (например, проверка интернет-эквайринга) не должны попадать в книгу доходов.

**Как пометить транзакцию как тестовую:**
1. В админке `/superadmin/accounting` раскройте «Детализация» в книге доходов.
2. Найдите тестовую транзакцию по дате/сумме.
3. Нажмите кнопку 🧪 справа в строке.
4. Строка станет жёлтой и выпадет из всех итогов (если чекбокс «Показать тестовые» выключен).

После этого в книге доходов этих сумм не будет. В банковской выписке T-Bank они тоже обычно отсутствуют, поскольку шли через тестовый терминал.
