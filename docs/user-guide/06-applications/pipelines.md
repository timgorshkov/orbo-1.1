# Воронки и статусы заявок

## Что такое воронка

Воронка (Pipeline) — это последовательность этапов, через которые проходит заявка. Каждый этап имеет:

- **Название** — например, «Новая», «На рассмотрении», «Одобрена»
- **Цвет** — для визуального различения в интерфейсе
- **Позиция** — порядок в воронке
- **Терминальное состояние** — является ли этап финальным (успех или отказ)

---

## Этапы по умолчанию

При создании воронки автоматически создаются стандартные этапы:

| Этап | Тип | Описание |
|------|-----|----------|
| Новая | Начальный | Заявка только что поступила |
| На рассмотрении | Промежуточный | Администратор изучает заявку |
| Одобрена | Терминальный (успех) | Заявка принята |
| Отклонена | Терминальный (отказ) | Заявка отклонена |

Вы можете добавить промежуточные этапы по своему усмотрению.

---

## Создание воронки

### Шаг 1. Откройте раздел «Заявки»

В левом меню выберите **«Заявки»**.

### Шаг 2. Нажмите «Создать воронку»

### Шаг 3. Заполните форму

| Поле | Описание |
|------|----------|
| Название | Например, «Заявки на вступление» |
| Тип | `join_request`, `service` или `custom` |
| Группа | Привязка к Telegram-группе (для join_request) |

### Шаг 4. Настройте этапы

После создания откройте настройки воронки, чтобы изменить или добавить этапы.

---

## Перемещение заявки

### Из интерфейса

1. Откройте заявку
2. Выберите новый этап из выпадающего списка
3. При необходимости добавьте заметку

### Терминальные этапы

Когда заявка перемещается в терминальный этап:

- **Успех (approve)** — для join_request заявитель одобряется в группу
- **Отказ (reject)** — заявка отклоняется

Автоматические действия (approve/reject в Telegram) выполняются при перемещении в терминальный этап.

---

## Настройки воронки

В настройках воронки можно:

- Переименовать воронку
- Изменить порядок и цвета этапов
- Удалить неиспользуемую воронку
