# Интеграция страниц Orbo в WordPress

## Структура файлов

```
docs/website-html/
├── styles.css          # Общие стили (дизайн-система)
├── icons.svg           # SVG-спрайт с иконками
├── home.html           # Главная страница (переделка orbo.ru)
├── product.html        # Страница "Продукт" (пункт меню)
├── events.html         # События и регистрации (подменю)
├── notifications.html  # Группы и уведомления (подменю)
├── crm.html            # CRM участников (подменю)
└── README.md           # Эта инструкция
```

---

## Шаг 1. Подключение стилей

### 1.1 Загрузка CSS
1. Скопируйте содержимое `styles.css`
2. В WordPress: **Внешний вид → Настроить → Дополнительные стили**
3. Вставьте весь CSS

**Или через файл темы:**
1. Загрузите `styles.css` в `/wp-content/themes/astra/` (или `/astra-child/`)
2. В `functions.php` добавьте:

```php
function orbo_enqueue_styles() {
    wp_enqueue_style('orbo-styles', get_template_directory_uri() . '/styles.css', array(), '1.0');
}
add_action('wp_enqueue_scripts', 'orbo_enqueue_styles');
```

### 1.2 Подключение шрифта Inter
В **Внешний вид → Настроить → Дополнительные стили** добавьте в начало:

```css
@import url('https://fonts.googleapis.com/css2?family=Inter:wght@400;500;600;700;800&display=swap');
```

## Шаг 2. SVG-иконки (уже сделано)

Файл `icons.svg` загружен в WordPress:
```
https://www.orbo.ru/wp-content/uploads/2025/12/icons.svg
```

Все HTML-файлы уже используют этот путь. Иконки вызываются так:
```html
<svg width="24" height="24"><use href="https://www.orbo.ru/wp-content/uploads/2025/12/icons.svg#icon-check"/></svg>
```

**Примечание:** Если иконки не отображаются, проверьте CORS-заголовки на сервере или используйте inline-вариант (вставить содержимое icons.svg в начало каждой страницы).

## Шаг 3. Создание страниц

**ВАЖНО:** Используйте файлы из папки `wordpress-content/` — они содержат только контент без обёрток `<!DOCTYPE>`, `<html>`, `<head>`, `<body>`.

### Правильный способ вставки HTML в WordPress:

**Вариант А: Блок "Произвольный HTML" (рекомендуется)**
1. В визуальном редакторе нажмите **+** (добавить блок)
2. Найдите блок **"Произвольный HTML"** (Custom HTML)
3. Вставьте содержимое из файла `wordpress-content/home-content.html`
4. Сохраните

**Вариант Б: Классический редактор**
1. Установите плагин **Classic Editor** (если нужен)
2. Переключитесь на вкладку **Текст** (не Визуальный)
3. Вставьте HTML

**НЕ РАБОТАЕТ:** "Три точки → Редактор кода" — это просмотр структуры блоков, WordPress добавит `<p>` и `<br>` теги.

### 3.1 Главная страница
1. **Страницы → Редактировать** главную страницу
2. Удалите все существующие блоки
3. Добавьте блок **"Произвольный HTML"**
4. Вставьте содержимое из `wordpress-content/home-content.html`
5. Сохраните

### 3.2 Страница "Продукт"
1. **Страницы → Добавить новую**
2. Заголовок: `Продукт`
3. URL: `/product/`
4. Добавьте блок **"Произвольный HTML"**
5. Вставьте содержимое из `wordpress-content/product-content.html`
6. Публикуйте

### 3.3 Подстраницы (events, notifications, crm)

**События:**
1. Заголовок: `События и регистрации`
2. URL: `/product/events/`
3. Родительская: `Продукт`
4. Вставьте HTML из `events.html`

**Уведомления:**
1. Заголовок: `Группы и уведомления`
2. URL: `/product/notifications/`
3. Родительская: `Продукт`
4. Вставьте HTML из `notifications.html`

**CRM:**
1. Заголовок: `CRM участников`
2. URL: `/product/crm/`
3. Родительская: `Продукт`
4. Вставьте HTML из `crm.html`

## Шаг 4. Настройка меню

1. **Внешний вид → Меню**
2. Создайте или выберите главное меню
3. Добавьте страницу **Продукт**
4. Перетащите подстраницы под "Продукт" с отступом (станут подменю):
   - События и регистрации
   - Группы и уведомления
   - CRM участников

```
Главная
Продукт
  └── События и регистрации
  └── Группы и уведомления
  └── CRM участников
О нас
Контакты
```

5. Сохраните меню

## Шаг 5. Настройка Astra для полноширинных страниц

Для каждой созданной страницы:

1. Откройте страницу на редактирование
2. В правой панели найдите **Astra Settings**
3. Установите:
   - **Sidebar**: No Sidebar
   - **Content Layout**: Full Width / Stretched
   - **Disable Title**: Yes (заголовок уже есть в HTML)
   - **Disable Header**: No (хедер с меню нужен)
   - **Disable Footer**: No

## Шаг 6. Замена плейсхолдеров скриншотов

В HTML есть блоки вида:
```html
<div class="screenshot-placeholder">
  <svg class="screenshot-placeholder__icon"><use href="icons.svg#icon-image"/></svg>
  <p class="screenshot-placeholder__text">СКРИНШОТ: Дашборд с метриками</p>
</div>
```

Когда скриншоты будут готовы, замените на:
```html
<img src="https://orbo.ru/wp-content/uploads/screenshot-dashboard.png" 
     alt="Дашборд с метриками активности" 
     class="showcase__image">
```

## Дополнительно: Адаптация header/footer

Существующие header и footer Astra могут конфликтовать со стилями страниц. Варианты решения:

### Вариант А: Использовать header/footer Astra
Убедитесь, что цветовая схема Astra соответствует дизайну:
- Основной цвет: `#8B5CF6` (фиолетовый)
- Цвет текста: `#1F2937`
- Фон: `#FFFFFF`

### Вариант Б: Скрыть и добавить свой
В Astra Settings страницы:
- Disable Header: Yes
- Disable Footer: Yes

Затем добавьте свой header/footer в HTML страницы.

## Поддержка

Если возникнут вопросы по интеграции, обращайтесь. Основные моменты:

1. **Стили не применяются** — проверьте, что CSS загружен (Ctrl+U → поиск "orbo" или класса)
2. **Иконки не отображаются** — используйте inline SVG вместо внешнего файла
3. **Страница сжата в узкую колонку** — настройте Content Layout: Full Width в Astra Settings
4. **Заголовок дублируется** — отключите Title в Astra Settings
